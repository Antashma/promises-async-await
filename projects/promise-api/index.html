Practice Problems: Promise API

Explanation

Each practice problem focuses on one specific method:

  1. Promise.all: Requires all promises to succeed.
  2. Promise.allSettled: Provides the status of each promise, regardless of the outcome.
  3. Promise.any: Resolves as soon as any one of the promises succeeds.
  4. Promise.race: Resolves or rejects as soon as any one of the promises settles.

These problems should help you understand how each method works and how to handle different scenarios using the Promise class.

APIs

  • User API: https://jsonplaceholder.typicode.com/users/1
  • Posts API: https://jsonplaceholder.typicode.com/posts?userId=1
  • Comments API: https://jsonplaceholder.typicode.com/comments?postId=1

Practice Problem 1: Promise.all

Scenario

You need to fetch user data, posts, and comments simultaneously. All requests must succeed for the operation to be considered successful.

Task

Fetch the user data, posts, and comments simultaneously using Promise.all. If any request fails, handle the error.

Practice Problem 2: Promise.allSettled

Scenario

You need to fetch user data, posts, and comments simultaneously. You need to know the status of each request, regardless of whether they succeed or fail.

Task

Fetch the user data, posts, and comments simultaneously using Promise.allSettled.Display the status of each request.

Practice Problem 3: Promise.any

Scenario

You need to fetch user data, posts, and comments simultaneously. You only need one request to succeed for the operation to be considered successful.

Task

Fetch the user data, posts, and comments simultaneously using Promise.any. If at least one request succeeds, display the result. If all requests fail, handle the aggregate error.

Practice Problem 4: Promise.race

Scenario

You need to fetch user data, posts, and comments simultaneously. You want to use the result of the first request to complete, whether it succeeds or fails.

Task

Fetch the user data, posts, and comments simultaneously using Promise.race. Display the result of the first request to complete, whether it is successful or not.
