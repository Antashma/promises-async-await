Practice Problem: Error Handling with Promises

This practice problem will help you understand how to work with Promises and handle errors effectively in a real-world scenario.

Scenario:

You are building a simple web application that fetches user data from a mock API and then fetches additional information about the user's GitHub profile. If any part of the process fails (e.g., network issues, invalid user data), you need to handle the errors appropriately and notify the user.

Steps:

  1. Fetch user data from a mock API endpoint: `https://jsonplaceholder.typicode.com/users/1`
  2. Extract the username from the fetched user data.
  3. Fetch the user's GitHub profile from the GitHub API: `https://api.github.com/users/{username}`
  4. Display the user's GitHub avatar on the webpage.
  5. Handle any errors that occur during the fetch operations and display an alert with an appropriate error message.

Requirements:

  • Use the fetch API to make the network requests.
  • Chain Promises to handle the asynchronous operations.
  • Implement proper error handling using .catch.
  • Ensure that the error handling works for both network errors and invalid data.

Tasks:

  1. Implement the fetch operations to get user data and GitHub profile.
  2. Properly handle and propagate errors using `.catch`.
  3. Display the GitHub avatar on the page if all fetch operations are successful.
  4. Test your implementation by intentionally causing errors (e.g., using an invalid URL).

Expected Outcome:

  • If both fetch operations succeed, the user's GitHub avatar should be displayed on the webpage.
  • If any fetch operation fails, an alert should display an appropriate error message.
